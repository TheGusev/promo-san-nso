# PHP резервный канал отправки лидов

Файл `lead.php` — резервный приёмник заявок. Если основной канал (Supabase edge function) не ответил за 6 секунд, фронтенд шлёт лид сюда.

## Установка на Beget (гордэз.рф)

1. Откройте файловый менеджер Beget или FTP.
2. В корне сайта `гордэз.рф` создайте папку `api`.
3. Залейте `lead.php` в `/api/lead.php`.
4. **ВАЖНО:** откройте файл и впишите свой токен бота:
   ```php
   $TELEGRAM_BOT_TOKEN = 'PASTE_YOUR_BOT_TOKEN_HERE';
   ```
   Токен возьмите у @BotFather (тот же, что в Supabase secret `TELEGRAM_BOT_TOKEN`).
5. Проверьте `chat_id`: сейчас `-5244841627` (группа DZN). Если изменится — поправьте.

## Проверка

GET-запрос должен вернуть ошибку метода:
```
curl https://гордэз.рф/api/lead.php
# {"ok":false,"error":"method"}
```

POST с тестовыми данными:
```
curl -X POST https://гордэз.рф/api/lead.php \
  -H "Content-Type: application/json" \
  -d '{"name":"Тест","phone":"9991234567","source":"test"}'
# {"ok":true,"channel":"php"}
```

В Telegram должно прийти сообщение с пометкой `🚨 НОВАЯ ЗАЯВКА (резерв PHP)`.

## Лог

Каждая попытка пишется в `/api/lead.log` рядом с PHP-файлом. Формат:
```
[2026-05-07T12:34:56+07:00] ip=1.2.3.4 phone=+79991234567 src=lp_klopy tg=ok
```

## Безопасность

- Honeypot проверка (поле `honeypot` или `website`).
- Rate-limit: 5 заявок/минуту с одного IP.
- Валидация телефона (10–11 цифр).
- Токен в коде — не выкладывайте файл в публичный git.
