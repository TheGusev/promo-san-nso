

# Починить кнопку «Все услуги» в Hero

## Проблема
Кнопка «Все услуги» в hero-секции главной страницы (`src/components/Hero.tsx`, строка 99) вызывает `scrollToAnchor('services')`, но на главной нет элемента с `id="services"` — секция услуг имеет `id="prices"`. Поэтому при клике ничего не происходит.

При этом везде на сайте (шапка, футер, RelatedLinks) «Все услуги» ведёт на отдельную страницу `/uslugi` со списком всех услуг — это логичное и ожидаемое поведение.

## Решение (1 файл — `src/components/Hero.tsx`)

### 1. Заменить скролл на навигацию
- Импортировать `useNavigate` из `react-router-dom`.
- В `handleCTAClick('secondary')` вместо `scrollToAnchor('services')` сделать `navigate('/uslugi')` — переход на полный каталог услуг.
- Первичная кнопка («Рассчитать стоимость») остаётся как есть — скроллит к калькулятору.

```tsx
const navigate = useNavigate();

const handleCTAClick = (type: 'primary' | 'secondary') => {
  if (type === 'primary') {
    scrollToAnchor('calculator');
  } else {
    navigate('/uslugi');
  }
};
```

## Что НЕ трогаем
- Секция `Services` (`id="prices"`) — оставляем как есть, это якорь для пункта меню «Цены».
- Шапка, футер, остальные ссылки «Все услуги» — уже работают корректно.
- Стили, аналитика, MVT-логика.

## Итог
1 файл, ~3 строки. Кнопка «Все услуги» в hero на главной начнёт открывать страницу `/uslugi` — единое поведение со всем сайтом.
