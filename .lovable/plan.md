

# Добавить фоновые фото на страницы объектов

## Проблема

Сейчас hero-секция страниц объектов (квартира, дом, склад и т.д.) показывает только иконку Building в круге. Нужно заменить иконку на реальные фото объектов.

## Решение

### 1. Скопировать загруженные фото в `public/images/objects/`

Маппинг фото по объектам:
- `kvartira.jpg` — фото многоэтажного ЖК (photo_2026-02-23_23-05-36.jpg)
- `dom.jpg` — коттеджный посёлок (photo_2026-02-23_23-05-29.jpg)
- `dacha.jpg` — обработка у дома (photo_2026-02-23_23-06-28.jpg)
- `uchastok.jpg` — обработка участка (photo_2026-02-23_23-06-31.jpg)
- `ofis.jpg` — обработка офиса (photo_2026-02-23_23-06-49.jpg)
- `restoran.jpg` — обработка в деревянном доме/помещении (photo_2026-02-23_23-06-08.jpg)
- `sklad.jpg` — складской комплекс (photo_2026-02-23_23-05-21.jpg)
- `default.jpg` — озонатор в комнате (photo_2026-02-23_23-05-53.jpg) — для объектов без своего фото

### 2. Добавить поле `heroImage` в `ObjectType` (src/data/objects.ts)

Добавить опциональное поле `heroImage?: string` в интерфейс и прописать пути к каждому объекту.

### 3. Обновить hero в `ObjectPage.tsx`

Заменить блок с иконкой Building (строки 75-82) на `<img>` с фото объекта. На мобилке — фото над текстом с закруглением и затемнением. На десктопе — справа в колонке.

### Файлы

| Файл | Действие |
|------|----------|
| `public/images/objects/*.jpg` | Скопировать 8 фото |
| `src/data/objects.ts` | Добавить `heroImage` поле |
| `src/pages/obekt/ObjectPage.tsx` | Заменить иконку на фото |
