

# Анализ проблемы конверсии мобильного калькулятора

## Что происходит сейчас (из кода и скриншота)

На мобилке форма разделена на **две физически разные части**:

1. **Карточка калькулятора** (в потоке страницы) — цена, тип объекта, площадь, услуга, тип клиента. Контактные поля **скрыты** (`hidden sm:block`, строка 692).

2. **Sticky-футер** (прибит к низу экрана, строки 793-845) — имя, телефон, кнопка «Заявка → 1 425 ₽».

Проблема: пользователь настраивает параметры в калькуляторе, видит цену, но **не понимает связь** между калькулятором и sticky-полоской внизу. Sticky-футер выглядит как отдельный элемент, а не продолжение формы. Между ними — секция «О компании», что ещё больше разрывает контекст.

На скриншоте видно: стрелки указывают на разрыв между калькулятором и формой ввода имени/телефона внизу.

## Решение: объединить форму в одну карточку

Убрать sticky-футер. Встроить поля «Имя» и «Телефон» прямо в карточку калькулятора на мобилке — сразу после параметров расчёта и перед кнопкой отправки. Поток становится линейным:

```text
┌─────────────────────────┐
│  💰 1 425 ₽  (-5%)      │  ← цена сразу видна
├─────────────────────────┤
│  Тип объекта  │ Площадь │
│  Услуга       │ Клиент  │
│  [Метод и периодичность]│
├─────────────────────────┤
│  Имя *        │ Телефон │  ← теперь прямо тут
├─────────────────────────┤
│  [Заявка → 1 425 ₽]    │  ← кнопка в карточке
│  Перезвоним за 5 минут  │
└─────────────────────────┘
```

## Изменения в файле

### `src/components/Calculator.tsx`

1. **Убрать `hidden sm:block`** с блока контактных данных (строка 692) — показывать имя/телефон/кнопку на всех устройствах внутри карточки.

2. **Удалить весь sticky-футер** (строки 793-845) — больше не нужен, форма едина.

3. **Убрать `pb-20`** с секции калькулятора (строка 510) — паддинг снизу для sticky больше не нужен.

4. **Убрать `animate-pulse`** с кнопки — пульсация раздражает и выглядит как баг.

Итог: одна форма, один поток, один контекст. Пользователь видит цену → вводит имя и телефон → нажимает кнопку. Всё в одной карточке без разрыва.
