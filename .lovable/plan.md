

# Обновить Telegram бот + chat_id и протестировать отправку с калькулятора

## Контекст
Логи `handle-lead` показывают свежую ошибку: `Telegram API error: 400 "Bad Request: chat not found"`. То есть текущий `TELEGRAM_BOT_TOKEN` и/или `TELEGRAM_CHAT_ID` в секретах больше не валидны — у вас новый бот и новая группа.

Код самой отправки (`supabase/functions/handle-lead/index.ts`, строки 269–321) рабочий, менять его не нужно. Также есть служебная функция `supabase/functions/test-telegram/index.ts` для быстрой диагностики.

## Шаги

### 1. Обновить секреты
Через `add_secret` запросить у вас два значения:
- **`TELEGRAM_BOT_TOKEN`** — токен нового бота от @BotFather (вид `1234567890:AAH...`).
- **`TELEGRAM_CHAT_ID`** — id новой группы (для супергруппы вид `-100xxxxxxxxxx`, для обычной `-xxxxxxxxx`).

Чтобы получить chat_id новой группы:
1. Добавить вашего нового бота в группу **админом** (право «Отправка сообщений» обязательно).
2. Написать в группе любое сообщение.
3. Открыть `https://api.telegram.org/bot<TOKEN>/getUpdates` в браузере — найти `"chat":{"id":-100…}`. Это нужное значение.

### 2. Прогнать `test-telegram`
Сразу после обновления секретов вызвать `test-telegram` через edge-функцию. Ожидание: в новой группе появляется сообщение «🧪 Тестовое сообщение… ✅ Telegram интеграция работает!». Если приходит — секреты валидны.

Если `test-telegram` вернёт `chat migrated to supergroup` с новым id — обновлю `TELEGRAM_CHAT_ID` на присланный Telegram'ом id и повторю тест.

### 3. Сквозной тест от калькулятора
Вызвать `handle-lead` с реалистичной payload, имитирующей заявку:
```json
{
  "name": "Тест Калькулятор",
  "phone": "+79991234567",
  "source": "website_calculator_simple",
  "service": "Дезинсекция",
  "object_type": "Квартира",
  "final_price": 2500,
  "session_id": "test-session-...",
  "form_start_time": <now - 5000>
}
```
Ожидание: в группу приходит сообщение «🆕 Новая заявка #...» со всеми полями.

### 4. Проверка логов
После каждого теста читать `edge_function_logs` для `handle-lead` и `test-telegram`. Подтвердить отсутствие ошибок 4xx от Telegram API.

### 5. Если всё ОК — обновить память проекта
Записать новый `chat_id` в `mem://technical/telegram-integration-chat-id-migration` (заменить старый `-1003211592275` на актуальный), чтобы в будущем не путаться.

## Что НЕ трогаем
- Код `SimpleCalculator.tsx` и `handle-lead/index.ts` — без изменений, проблема только в секретах.
- Структура лида, валидации, rate-limit, Schema.org, остальные функции.

## Итог
0 правок в коде. Обновляются 2 секрета → 2 теста edge-функций → проверка логов → обновление памяти. Время — 2–3 минуты после получения от вас токена и chat_id.
