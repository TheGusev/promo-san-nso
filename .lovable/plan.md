
# План глобальной чистки текстов от артефактов ИИ-генерации

## Обнаруженные проблемы

### 1. Служебные LSI-комментарии (видны пользователям!)

Во всех контентных файлах присутствуют комментарии вида `/* LSI: описание */`, которые рендерятся как часть текста:

**Примеры из `pestContent.ts`:**
```
appearance: `... /* LSI: внешний вид клопа */`
habitat: `... /* LSI: где живут клопы в квартире */`
danger: `... /* LSI: чем опасны клопы для человека */`
```

**Примеры из `serviceContent.ts`:**
```
/* LSI: дезинсекция новосибирск, уничтожение насекомых */
```

**Примеры из `objectContent.ts`:**
```
/* LSI: обработка квартиры от насекомых */
```

**Примеры из `districtContent.ts`:**
```
/* LSI: дезинсекция Центральный район Новосибирск */
```

### 2. Ошибки в согласовании заголовков

В компоненте `PestPage.tsx` (строка 142) есть попытка согласования:
```tsx
<h2>Чем опасен{pest.category === "rodents" ? "ы" : ""} {pest.namePlural}</h2>
```

Проблема: логика неполная — добавляет "ы" только для грызунов, но не для насекомых во множественном числе. Например:
- "Чем опасен клопы" ❌ (должно быть "Чем опасны клопы")
- "Чем опасен тараканы" ❌ (должно быть "Чем опасны тараканы")
- "Чем опасен блохи" ❌ (должно быть "Чем опасны блохи")

В данных `pests.ts` все вредители имеют `namePlural` во множественном числе, поэтому заголовок должен быть "Чем опасны" для всех.

### 3. Инструктивные комментарии в начале файлов

В начале файлов есть инструкции, которые не влияют на вывод, но захламляют код:
```ts
// Для LSI-ключей используйте места, помеченные комментариями /* LSI: описание */
```

## План исправлений

### Файл 1: `src/data/pestContent.ts` (766 строк)

Удалить все вхождения `/* LSI: ... */` из текстовых полей:

| Поле | Примеры LSI-комментариев для удаления |
|------|--------------------------------------|
| `appearance` | `/* LSI: внешний вид клопа */`, `/* LSI: как выглядит таракан рыжий и чёрный */`, `/* LSI: как выглядит блоха фото */` |
| `habitat` | `/* LSI: где живут клопы в квартире */`, `/* LSI: где прячутся тараканы в квартире */`, `/* LSI: откуда берутся блохи в квартире без животных */` |
| `danger` | `/* LSI: чем опасны клопы для человека */`, `/* LSI: чем опасны тараканы для здоровья */`, `/* LSI: чем опасны укусы блох для человека */` |
| `treatmentProcess` | `/* LSI: как травят клопов профессионалы */`, `/* LSI: как травят тараканов профессионалы */` |

Также удалить инструктивный комментарий в начале файла (строка 2).

### Файл 2: `src/data/serviceContent.ts` (729 строк)

Удалить LSI-комментарии из:
- Поле `fullDescription` (например: `/* LSI: дезинсекция новосибирск, уничтожение насекомых, обработка от тараканов клопов */`)
- Инструктивный комментарий в начале файла (строка 3)

### Файл 3: `src/data/objectContent.ts` (619 строк)

Удалить LSI-комментарии из:
- Поле `intro` (например: `/* LSI: обработка квартиры от насекомых */`, `/* LSI: обработка частного дома от насекомых и грызунов */`)
- Инструктивный комментарий в начале файла (строка 2)

### Файл 4: `src/data/districtContent.ts` (453 строки)

Удалить LSI-комментарии из:
- Поле `intro` (например: `/* LSI: дезинсекция Центральный район Новосибирск */`, `/* LSI: СЭС Ленинский район Новосибирск */`)
- Инструктивный комментарий в начале файла (строка 2)

### Файл 5: `src/pages/vreditel/PestPage.tsx`

Исправить заголовок "Чем опасен/опасны" (строка 141-143):

**Было:**
```tsx
<h2 className="text-2xl font-bold md:text-3xl">
  Чем опасен{pest.category === "rodents" ? "ы" : ""} {pest.namePlural}
</h2>
```

**Станет:**
```tsx
<h2 className="text-2xl font-bold md:text-3xl">
  Чем опасны {pest.namePlural}
</h2>
```

Логика: все вредители в системе имеют `namePlural` во множественном числе (клопы, тараканы, блохи, крысы, мыши и т.д.), поэтому заголовок всегда должен быть "Чем опасны".

Также удалить комментарий в строке 52:
```tsx
{/* LSI: место для ключей в H1 и подзаголовке */}
```

## Техническая реализация

Используем инструмент `lov-line-replace` для точечных изменений в каждом файле:

1. Найти все вхождения паттерна `/* LSI: ... */`
2. Удалить их, сохраняя окружающий текст
3. Исправить заголовок в PestPage.tsx
4. Удалить инструктивные комментарии в начале файлов

---

## Что НЕ меняем

- URL, slug'и, внутренние ссылки — без изменений
- Уровни заголовков (H2, H3) — без изменений
- Структуру блоков и компонентов — без изменений
- Фактический контент (болезни, симптомы, рекомендации) — сохраняем
- HTML-комментарии в JSX, не влияющие на вывод — оставляем
