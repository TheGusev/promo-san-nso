

# Исправление ошибки 404 для Яндекс Вебмастера

## Проблема

Яндекс Вебмастер сообщает: **«Некорректно настроено отображение несуществующих файлов и страниц»**. Это значит, что при обращении к несуществующему URL сервер возвращает HTTP 200 вместо 404 — типичная проблема SPA (React).

Дополнительно: текущая страница 404 на английском языке ("Oops! Page not found"), а сайт полностью русскоязычный.

## Решение

### 1. Создать статический `public/404.html`

Хостинг Lovable (как и большинство SPA-хостингов) при наличии файла `public/404.html` будет отдавать его с правильным HTTP-кодом 404 для несуществующих статических файлов. Файл будет содержать:
- Корректный HTTP-ответ для поисковых роботов
- Мета-тег `<meta name="robots" content="noindex">` (чтобы 404-страницы не индексировались)
- Русскоязычный текст
- Ссылку на главную страницу
- Авторедирект на React-приложение для SPA-маршрутов

### 2. Обновить `src/pages/NotFound.tsx`

- Перевести на русский язык
- Добавить `<meta name="robots" content="noindex">` через `useEffect`
- Добавить `document.title` для страницы 404
- Улучшить дизайн (иконка, брендинг)

### Файлы для изменения

| Файл | Действие |
|------|----------|
| `public/404.html` | Создать — статическая 404-страница с HTTP 404 |
| `src/pages/NotFound.tsx` | Обновить — русский язык, noindex, улучшенный дизайн |
