

# Калькулятор: заменить плитки на выпадающие списки

## Что меняем
Сейчас на шаге 1 показано 12 карточек-плиток вредителей сразу — занимает много места и выглядит «громко». Пользователь хочет компактный селект: нажал → выпал список → выбрал → перешёл дальше.

## Новый поток (тот же 3-шаговый, но компактный)

```text
┌─────────────────────────────────────┐
│  Шаг 1 из 3                         │
│  Кто вас беспокоит?                 │
│  ┌─────────────────────────────┐    │
│  │ Выберите вредителя       ▾ │    │  ← Select (закрытый)
│  └─────────────────────────────┘    │
│  [Далее] (disabled пока не выбрано) │
└─────────────────────────────────────┘

После выбора (например, «Клопы»):
┌─────────────────────────────────────┐
│  Шаг 2 из 3                         │
│  Где провести обработку?            │
│  ┌─────────────────────────────┐    │
│  │ Квартира 2-комн до 60 м²  ▾│    │  ← Select только с доступными
│  └─────────────────────────────┘    │
│  [Назад]  [Далее]                   │
└─────────────────────────────────────┘

Шаг 3 — без изменений: цена + телефон + кнопки.
```

## Изменения в `src/components/SimpleCalculator.tsx`

### 1. Шаг 1 — выпадающий список вредителей
Вместо сетки 12 карточек:
- Один компонент `<Select>` (shadcn) с группировкой по категориям через `<SelectGroup>` + `<SelectLabel>`:
  - **Насекомые**: Клопы, Тараканы, Блохи, Муравьи, Моль, Кожеед, Осы, Комары, Клещи
  - **Грызуны**: Крысы, Мыши
  - **Другое**: Кроты
- В каждом `<SelectItem>` — название вредителя + маленькая подпись через `text-muted-foreground` (опционально).
- Под селектом — кнопка **«Далее»**, активируется после выбора и явно переключает на шаг 2 (`setStep(2)`).
- Если у выбранного вредителя только одно доступное место — пропускаем шаг 2 автоматически (как сейчас).

### 2. Шаг 2 — выпадающий список мест
Сейчас тоже плитки — заменить на `<Select>`:
- Опции = `getAvailablePlaces(pestSlug)` (динамический список 1–7 пунктов).
- Подписи: «Квартира 1-комн до 40 м²», «Квартира 2-комн до 60 м²», «Дом / коттедж», «Участок», «Коммерческое помещение» и т.д.
- Кнопки **«Назад»** и **«Далее»** (Далее активна после выбора, ведёт на шаг 3).

### 3. Шаг 3 — без изменений
Анимированная цена + поле телефона + чекбокс согласия + «Вызвать мастера» / «Позвонить». Логика отправки лида, аналитика (`calc_calculate`, `lead_submit`), офлайн-очередь — всё как сейчас.

### 4. Прогресс-индикатор
Оставить вверху карточки строку «Шаг X из 3» + тонкий progress-bar (если уже есть — не трогаем; если нет — добавим простой `<div>` с заливкой 33% / 66% / 100%).

## Поведение
- Селект на мобиле раскрывается на весь экран (нативное поведение Radix Select на тач-устройствах) — удобно пальцем.
- Тач-таргеты ≥ 56px (`h-14` для триггера селекта и кнопок «Далее/Назад»).
- При смене вредителя на шаге 1 — сбрасываем выбранное место (`placeKey = null`), чтобы не унаследовать невалидное.
- Кнопка «Далее» disabled, пока значение не выбрано (визуально приглушённая).

## Что НЕ трогаем
- `src/data/pests.ts`, `src/data/calculatorPricing.ts` — без изменений.
- Цены, прайс-матрица, маппинг `place_key → object_type`, поля лида в `handle-lead` — без изменений.
- Якорь `id="calculator"` и offset-скролл — без изменений.
- `Calculator.tsx` (старый) — не трогаем.
