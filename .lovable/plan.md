
# План исправления бага: прокрутка наверх при переходе между страницами

## Проблема

При навигации между страницами (услуги, вредители, объекты, районы, programmatic) в SPA на React Router скролл сохраняет предыдущую позицию. Пользователь видит середину новой страницы вместо её начала (H1/hero-блок).

Это особенно критично на мобильных устройствах — при выборе района в блоке «Другие районы» страница не прокручивается к заголовку.

---

## Причина

React Router (v6) не сбрасывает скролл автоматически при переходе между страницами, в отличие от традиционных сайтов с полной перезагрузкой. Браузер сохраняет scroll position для текущего документа.

---

## Решение

Создать глобальный компонент `ScrollToTop`, который:
1. Подписывается на изменения `pathname` через хук `useLocation()`
2. При каждом изменении пути вызывает `window.scrollTo(0, 0)`
3. Подключается один раз в корневом роутере (`App.tsx`)

---

## Файлы для изменения

| Файл | Действие |
|------|----------|
| `src/components/ScrollToTop.tsx` | Создать |
| `src/App.tsx` | Подключить компонент внутри BrowserRouter |

---

## Реализация

### 1. Создать `src/components/ScrollToTop.tsx`

```typescript
import { useEffect } from "react";
import { useLocation } from "react-router-dom";

export function ScrollToTop() {
  const { pathname } = useLocation();

  useEffect(() => {
    // Прокрутка в самый верх при смене маршрута
    window.scrollTo({
      top: 0,
      left: 0,
      behavior: "instant" // Мгновенная прокрутка без анимации
    });
  }, [pathname]);

  return null; // Компонент ничего не рендерит
}
```

**Особенности:**
- `behavior: "instant"` — мгновенная прокрутка без плавной анимации (чтобы пользователь сразу видел начало страницы)
- Компонент возвращает `null` — он «невидимый», только выполняет side-effect
- Срабатывает при каждом изменении `pathname` (но не при изменении query-параметров или hash)

### 2. Обновить `src/App.tsx`

Подключить `ScrollToTop` ВНУТРИ `<BrowserRouter>`, но ДО `<Routes>`:

```diff
+ import { ScrollToTop } from "@/components/ScrollToTop";

// ...

<BrowserRouter>
+   <ScrollToTop />
    <Routes>
      {/* ... все маршруты ... */}
    </Routes>
</BrowserRouter>
```

**Важно:**
- Компонент должен быть внутри `<BrowserRouter>`, так как `useLocation()` требует контекста роутера
- Размещаем ДО `<Routes>`, чтобы прокрутка срабатывала до рендеринга новой страницы

## Дополнительно: исключения (опционально)

Если в будущем понадобится сохранять скролл при переходах по якорям (`#section`), можно расширить логику:

```typescript
useEffect(() => {
  // Если есть hash (#section) — не прокручивать в top
  if (!location.hash) {
    window.scrollTo({ top: 0, left: 0, behavior: "instant" });
  }
}, [pathname]);
```

Но для текущей задачи это не требуется.

---

## Результат

- При любом клике по внутренним ссылкам страница открывается с начала
- Улучшается UX на мобильных устройствах
- Работает глобально для всех страниц без дублирования кода
- Минимальные изменения: 1 новый файл + 2 строки в App.tsx
