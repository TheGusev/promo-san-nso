

# Автоматическая тема (светлая/тёмная) по системным настройкам

## Что есть сейчас
- `ThemeProvider` из `next-themes` уже подключён в `App.tsx` с `defaultTheme="light"` и `enableSystem` (т.е. техническая поддержка системной темы есть, но по умолчанию принудительно ставится светлая).
- Компонент `ThemeToggle` (Sun/Moon) переключает между `light` ↔ `dark` вручную, **но он нигде не используется в актуальной шапке `SiteHeader`** — только в старом `Header.tsx`, который не активен на сайте.
- В `index.css` определены оба набора токенов (`:root` и `.dark`) — визуально темы готовы.

## Что меняем

### 1. `src/App.tsx` — включить системную тему по умолчанию
```tsx
<ThemeProvider attribute="class" defaultTheme="system" enableSystem disableTransitionOnChange>
```
- `defaultTheme="system"` — при первом заходе тема берётся из ОС/браузера (`prefers-color-scheme`).
- Если пользователь вручную выбрал свет/тьму — выбор сохраняется в `localStorage` и больше не перетирается системой (стандартное поведение `next-themes`).
- `disableTransitionOnChange` — убирает мигание цветов при смене темы.

### 2. `src/components/layout/SiteHeader.tsx` — добавить кнопку переключения
Вставить `<ThemeToggle />` рядом с CTA (телефон/«Рассчитать») и в мобильном бургер-меню. Это даёт пользователю возможность вручную переопределить системную тему, если он захочет.

### 3. `src/components/ThemeToggle.tsx` — небольшое улучшение
Добавить третье состояние «Авто (как в системе)» через `setTheme('system')` — циклическое переключение `system → light → dark → system`. Иконка:
- system → монитор (`Monitor` из lucide)
- light → солнце
- dark → луна

Tooltip-подсказка «Тема: системная / светлая / тёмная».

### 4. `index.html` — `color-scheme` мета
Добавить `<meta name="color-scheme" content="light dark">` в `<head>`, чтобы браузер корректно красил скроллбары и нативные элементы под текущую тему до загрузки JS.

### 5. Анти-flash скрипт
`next-themes` сам вставляет инлайн-скрипт перед гидрацией — никаких ручных правок не нужно. Просто проверим, что `attribute="class"` и `enableSystem` стоят (см. п.1).

## Что НЕ трогаем
- HSL-токены в `:root` и `.dark` — обе темы уже отрисованы.
- Логику компонентов, цветовую систему, контрастность — всё через CSS-переменные.
- `Header.tsx` (старый, не используется) — оставляем как есть.

## Итог
- При первом заходе сайт автоматически подхватит тему мобильного устройства / ОС.
- Кнопка в шапке позволит вручную переключить на светлую / тёмную / снова авто.
- 4 файла к правке: `App.tsx`, `SiteHeader.tsx`, `ThemeToggle.tsx`, `index.html`.
