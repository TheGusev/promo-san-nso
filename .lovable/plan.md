
# План создания унифицированного HeroService компонента

## Проблема

На сайте СЭС «Гордез» сейчас используются два разных первых экрана:
- **Главная страница**: градиентный Hero с общим заголовком «Санитарные решения с гарантией результата»
- **Страницы услуг**: отдельный hero-блок с заголовком услуги, ценой и кнопками

На мобильных устройствах это создаёт ощущение «двух разных сайтов» из-за разных стилей и структуры.

---

## Решение

Создать единый компонент `HeroService` для всех страниц услуг с чёткой структурой, который будет:
- Принимать данные услуги (заголовок, описание, цена)
- Иметь встроенную форму заявки
- Адаптироваться для мобильных устройств

## Изменения в файлах

### 1. Создать `src/components/HeroService.tsx`

Новый компонент с пропсами:
```typescript
interface HeroServiceProps {
  title: string;              // "Дезинфекция"
  heading: string;            // "Дезинфекция в Новосибирске"
  description: string;        // Подзаголовок
  priceFrom: number;          // 1500
  icon: LucideIcon;           // Shield, Bug, Rat...
  serviceSlug: string;        // Для формы
}
```

Компонент содержит:
- Хлебные крошки (`Breadcrumbs`)
- Плашка услуги с иконкой
- H1 заголовок
- Описание (2-3 строки)
- Кнопка «Вызвать специалиста» (scrolls to form или tel:)
- Кликабельный телефон
- Строка «Работаем 24/7»
- Карточка цены
- Компактная форма: Имя, Телефон, кнопка «Заявка → цена»

### 2. Обновить `src/components/templates/ServicePageTemplate.tsx`

- Удалить текущую hero-секцию (строки 48-108)
- Импортировать и использовать `HeroService`
- Передать данные услуги в новый компонент

### 3. Обновить главную страницу `src/pages/Index.tsx`

- Оставить текущий `Hero.tsx` для главной страницы (он про общий бренд)
- Унифицировать хедер: заменить `Header` на `SiteHeader`

### 4. Удалить устаревшие компоненты

- `src/components/ServicePageLayout.tsx` — больше не нужен (используется только для редиректов)
- Удалить импорты старых страниц услуг в `App.tsx` (Dezinfeksiya.tsx и т.д.)

## Техническая реализация

### Данные для HeroService

Использовать существующий массив `services` из `src/data/services.ts`:
```typescript
{
  slug: "dezinfeksiya",
  name: "Дезинфекция",
  description: "Уничтожение вирусов...",
  priceFrom: 1500,
  icon: Shield,
}
```

### Форма в HeroService

Упрощённая форма с отправкой в существующий `handle-lead`:
- Имя (обязательно)
- Телефон (с маской)
- Кнопка «Заявка → {price} ₽»
- При отправке: вызов `supabase.functions.invoke('handle-lead')`

### Мобильная адаптация (Mobile-first)

```typescript
<section className="py-6 md:py-12">
  <div className="container px-4">
    {/* Breadcrumbs */}
    <Breadcrumbs items={[...]} />
    
    <div className="mt-4 grid gap-6 lg:grid-cols-2 lg:items-start">
      {/* Левая колонка: инфо */}
      <div className="space-y-4">
        <Badge icon={Icon}>{title}</Badge>
        <h1>{heading}</h1>
        <p>{description}</p>
        <div className="flex flex-col sm:flex-row gap-3">
          <Button>Вызвать специалиста</Button>
          <a href="tel:...">📞 Телефон</a>
        </div>
        <p>⏰ Работаем 24/7</p>
      </div>
      
      {/* Правая колонка: цена + форма */}
      <div className="space-y-4">
        <PriceCard price={priceFrom} />
        <QuickForm serviceSlug={serviceSlug} />
      </div>
    </div>
  </div>
</section>
```
