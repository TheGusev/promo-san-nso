

# Простой калькулятор в 3 шага

## Цель
Заменить текущий 6-параметрный калькулятор простым шаговым визардом: **Вредитель → Место → Цена + заявка**. Без шума, без коэффициентов в UI, цены строго совпадают с прайс-листом на сайте.

## Поток (3 экрана внутри одной карточки)

```text
ШАГ 1 / 3                  ШАГ 2 / 3                  ШАГ 3 / 3
┌──────────────┐           ┌──────────────┐           ┌──────────────────┐
│ Кто беспокоит│           │ Где проблема │           │   1 800 ₽        │
│              │           │              │           │ Клопы · 2-комн   │
│ 🛏 Клопы     │     →     │ 🏠 Квартира  │     →     │                  │
│ 🪳 Тараканы  │           │   1-комн     │           │ ☎ +7 (…) ___-…  │
│ 🐀 Крысы     │           │   2-комн     │           │ ☑ Согласен на ОД │
│ … 12 видов   │           │   3-комн     │           │                  │
│              │           │   4+ комн    │           │ [ Вызвать масте..│
│              │           │ 🏡 Дом       │           │  ── или ──       │
│              │           │ 🌳 Участок   │           │  📞 Позвонить    │
│              │           │ 🏢 Офис/Кафе │           │                  │
└──────────────┘           └──────────────┘           └──────────────────┘
   [крупные карточки]         [крупные карточки]        [крупная цифра + 1 поле]
```

Между шагами — кнопка «Назад» (стрелка) сверху и прогресс-полоска `1 / 3`.

## Шаг 1 — Вредитель (12 карточек из `src/data/pests.ts`)

Сетка 3×4 (моб. 2×6) с иконкой + названием. Используем существующий массив `pests`. Категория «кроты» автоматически разрешает только «Участок» на шаге 2 (фильтрация мест).

| Карточка | Иконка |
|---|---|
| Клопы, Тараканы, Блохи, Муравьи, Моль, Кожеед, Осы, Комары, Клещи, Крысы, Мыши, Кроты | Lucide (`Bug`, `Rat`, `Spider`, `Worm` и т.п.) |

## Шаг 2 — Место (фильтруется по вредителю)

| Место | Доступно для |
|---|---|
| Квартира 1-комн (до 40 м²) | все, кроме кротов |
| Квартира 2-комн (до 60 м²) | все, кроме кротов |
| Квартира 3-комн (до 90 м²) | все, кроме кротов |
| Квартира 4+ комн (от 90 м²) | все, кроме кротов |
| Частный дом | все, кроме кротов |
| Участок / двор | клещи, комары, осы, муравьи, **кроты**, мыши, крысы |
| Офис / кафе / магазин | все, кроме кротов |

Для кротов на шаге 1 шаг 2 сразу показывает только «Участок» — пользователь не запутается.

## Шаг 3 — Цена и заявка

Большая цифра цены (анимированная, `useAnimatedNumber`) + строка «{Вредитель} · {Место}». Под ней:
- 1 поле — телефон (`PhoneInput`, маска `+7 …`)
- Чекбокс согласия на обработку ПД (обязательный, ссылка на `/privacy`)
- Кнопка **«Вызвать мастера»** (primary, крупная)
- Альтернатива: **«📞 Позвонить 8 (383) 312-23-30»** (secondary, `tel:+73833122330`)

Имя/email **не запрашиваем** — менеджер уточнит при звонке. Это и есть «3 клика».

## Прайс-матрица (фиксированная, совпадает с PriceTable и страницами услуг)

Цены берутся из лукап-таблицы `[вредитель → место]`, никаких коэффициентов в рантайме. Источник правды — новый файл `src/data/calculatorPricing.ts`.

| Место → / Услуга ↓ | 1-комн | 2-комн | 3-комн | 4+ комн | Дом | Участок | Офис/Кафе |
|---|---|---|---|---|---|---|---|
| Дезинсекция (клопы, тараканы, блохи, муравьи, моль, кожеед) | 1500 | 1800 | 2100 | 2500 | 2500 | 2500 | от 2500 |
| Осы (удаление гнезда) | — | — | — | — | 2000 | 2000 | 2000 |
| Клещи / комары (участок) | — | — | — | — | — | 2500 | — |
| Дератизация (крысы, мыши) | 1800 | 2000 | 2300 | 2500 | 2500 | 2500 | от 2500 |
| Кроты | — | — | — | — | — | 2500 | — |

Цены ≤ 2500 ₽ — выше существующего потолка 2000 ₽ по жилью только для коммерции/участков, что согласовано с публичными прайсами на странице услуг. Для коммерции и больших площадей показываем «**от 2500 ₽**» с подписью «Точную цену уточнит менеджер».

## Технические детали

- **Поля заявки в `handle-lead`**: `name = ""` (пусто, не обязательно на сервере — проверить и при необходимости разрешить пустое имя в схеме), `phone`, `service` (выводится из вредителя через `pest.relatedServices[0]`), `object_type` (квартира/дом/участок/офис), `final_price` (из лукапа), `source: 'website_calculator_simple'`.
- **Аналитика**: те же цели через `trackGoal` — `calc_calculate` после шага 2 (есть цена), `lead_submit` после отправки, `phone_click` для альтернативной кнопки. Никаких новых целей — составная `all_conversions` продолжает работать.
- **Валидация**: только телефон (regex `+7\d{10}`) + обязательный чекбокс согласия. Имя в `calculatorLeadSchema` нужно сделать `optional` или передавать дефолт `"Клиент"`.
- **MVT/трекинг**: `variantId`, `impressionId`, UTM, `intent` пробрасываются как сейчас (`getTrackingContext`, `useABTest`).
- **Анимации цены**: `useAnimatedNumber` уже есть.
- **Мобильная вёрстка**: edge-to-edge, крупные тач-таргеты ≥ 56px, прогресс-полоска сверху.

## Что пропадёт из UI
Метод обработки (туман/точечная), периодичность (разово/абонемент), тип клиента (физ/юр), площадь в м², поле email, поле имени, расширенные настройки. Всё это менеджер уточнит по телефону — это и есть смысл «простоты».

## Совместимость с прайсом
Единственный источник правды — новый `calculatorPricing.ts`. В отдельной задаче можно подвязать к нему и `PriceTable.tsx`, но в этой итерации просто **синхронизируем числа вручную** с текущим `priceData` из `PriceTable.tsx` (1500/1800/2100/2500), чтобы расхождений не было.
